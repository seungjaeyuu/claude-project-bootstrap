# v0.3.0 프로젝트 라이프사이클 프레임워크

> 이론 근거: SDLC (ISO/IEC 12207) + PMBOK 하이브리드.
> 강제가 아닌 프레임워크 — "무엇을 잊지 않았나" 체크 용도.

---

## 6단계 모델

```
 ①기획  →  ②설계  →  ③개발  →  ④테스트  →  ⑤출시  →  ⑥운영
Planning   Design    Dev      Testing    Release   Ops
                                          ├─준비 (5a)
                                          └─배포 (5b)
              ◄──────── 반복 (Iterative) ────────►
```

---

## 단계별 상세

### ① 기획 (Planning)

**SDLC 매핑**: Planning + Requirements Analysis

| 활동 | 설명 |
|---|---|
| 요구사항 문서화 | 기능·비기능 요구사항 정리 |
| 기술 타당성 검토 | 기술 스택, 외부 의존성 실현 가능성 |
| 일정·리소스 추정 | 마일스톤, 인력, 비용 |

**플러그인 지원**: 없음 (도메인 종속적)

---

### ② 설계 (Design)

**SDLC 매핑**: System Design + Architecture

| 활동 | 설명 |
|---|---|
| 아키텍처 결정 기록 | 기술 스택, 데이터 모델, API 설계 |
| UI/UX 설계 | 와이어프레임, 디자인 시스템 |
| 프로젝트 초기화 | 스캐폴드, 환경 설정 |

**플러그인 지원**:
- `/init` — 프로젝트 스캐폴드
- 추천 플러그인: `figma` (Figma 연동 시)

---

### ③ 개발 (Development)

**SDLC 매핑**: Implementation

| 활동 | 설명 |
|---|---|
| 코드 작성 | 기능 구현 |
| 코드 리뷰 | PR 리뷰, 품질 확보 |
| 리팩토링 | 기술 부채 관리 |

**플러그인 지원**:
- RULES 자동 트리거 (CLAUDE.md §3)
- pre-commit hook 자동 검증
- `/audit --quality` — 코드 품질 점검
- 추천 플러그인: `superpowers` (상시), `code-simplifier` (리팩토링 시)

---

### ④ 테스트 (Testing)

**SDLC 매핑**: Testing + Verification & Validation

| 활동 | 설명 |
|---|---|
| 단위 테스트 | 함수·모듈 레벨 검증 |
| 통합 테스트 | 컴포넌트 간 연동 검증 |
| E2E 테스트 | 사용자 시나리오 전체 흐름 |
| 성능·부하 테스트 | 응답 시간, 동시 접속 (해당 시) |
| 접근성 검증 | AX identifier, 스크린 리더 (해당 시) |
| 품질 평가 | 코드 품질 리뷰, 컨텍스트 최적화 |

**플러그인 지원**:
- `/audit --baseline` — 베이스라인 리뷰 (Git diff 기반)
- `/audit --context` — 컨텍스트 최적화 점검
- `docs/test/TESTING_FRAMEWORK.md` — E2E 규약

---

### ⑤ 출시 (Release)

**SDLC 매핑**: Deployment

두 서브페이즈로 구성:

#### 5a. 출시 준비 (Pre-release)

| 활동 | 설명 |
|---|---|
| 보안 감사 | 시크릿 노출, 권한, 취약점 점검 |
| 법적 문서 | 개인정보처리방침, 이용약관 작성/검토 |
| 국제화 (i18n) | 지원 언어 점검, 미번역 항목 확인 |
| 버전 확정 | semver 버전 결정, CHANGELOG 업데이트 |
| 빌드번호 확인 | 최종 빌드번호 정합성 검증 |
| 앱스토어 메타데이터 | 스크린샷, 설명, 키워드 (모바일 해당 시) |

**플러그인 지원**:
- `/release` — 보안·법적·버전·i18n·테스트 일괄 체크
- 추천 플러그인: `security-guidance` (보안 감사 시 활성화)

#### 5b. 배포 (Deploy)

| 활동 | 설명 |
|---|---|
| 빌드·서명·업로드 | 플랫폼별 배포 파이프라인 |
| 릴리스 태그 | `git tag vX.Y.Z` |
| 배포 후 스모크 테스트 | 핵심 기능 동작 확인 |

**플러그인 지원**:
- 추천 플러그인: `vercel` (웹 배포 시)
- Firebase predeploy hook (Firebase 사용 시)

---

### ⑥ 운영 (Operations)

**SDLC 매핑**: Maintenance

| 활동 | 설명 |
|---|---|
| 모니터링 | 에러 추적, 성능 모니터링 |
| 버그 수정 | 사용자 보고 대응 |
| 반복 개선 | 다음 사이클 기획 연결 |
| 회고 | 프로세스 개선점 도출 |

**플러그인 지원**:
- `/retro-check` (v0.4.0 예정)
- TASK.md 백로그 업데이트

## /guide 단계 감지 휴리스틱

| 신호 | 추정 단계 |
|---|---|
| CLAUDE.md 없음 또는 소스 파일 0개 | ① 기획 / ② 설계 |
| 최근 커밋에 src/app 코드 변경 다수 | ③ 개발 |
| 최근 커밋에 test 파일 변경 다수 | ④ 테스트 |
| CHANGELOG 변경 또는 버전 파일 변경 | ⑤ 출시 |
| git tag가 최근 커밋에 존재 | ⑥ 운영 |

"추정"이므로 강제하지 않음. 참고 수준 안내.
