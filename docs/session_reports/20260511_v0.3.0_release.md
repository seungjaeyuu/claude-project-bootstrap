## 2. 빌드번호 동기화 수정 — 상세

### 2.1 배경 (사고 이력)

XcodeGen 기반 iOS 프로젝트에서 `project.yml`의 `CURRENT_PROJECT_VERSION`을 단일 원천(SSOT)으로 사용. 두 가지 문제가 반복 발생:

| 문제 | 원인 | 영향 |
|---|---|---|
| **이중 증가** | CLAUDE.md "수동 +1" 지시 + hook 자동 +1 | 한 커밋에 +2 (199→201) |
| **xcodeproj 미동기화** | hook이 `project.yml`만 변경, `xcodegen generate` 미실행 | Xcode/앱에 구 빌드번호 표시 |

### 2.2 수정 파일 (5개)

#### (1) `scripts/pre-commit-framework.sh` §(6) — 개선

**변경 전**: `$ROOT/iOS/project.yml` 단일 경로만 탐색, sed 치환 + staging만 수행

**변경 후**:
- 다중 경로 탐색: `iOS/project.yml` → `project.yml` → `app/project.yml` (모노레포 대응)
- `xcodegen generate --quiet` 실행 (XcodeGen 설치 시)
- 재생성된 `.xcodeproj` 디렉토리 자동 감지 + staging
- 헤더 주석에 "🚫 수동 증가 금지" 경고 추가
- 검사 항목 목록에 §(6) 설명 추가

핵심 코드:
```bash
PROJYML=""
for candidate in "$ROOT/iOS/project.yml" "$ROOT/project.yml" "$ROOT/app/project.yml"; do
  if [ -f "$candidate" ]; then PROJYML="$candidate"; break; fi
done
# ... increment + xcodegen generate + .xcodeproj staging
```

#### (2) `scripts/post-merge.sh` — 신규

- merge 후 `project.yml` 변경 감지 시 `.xcodeproj` 자동 재생성
- 빌드번호 증가 없음 (pre-commit hook과의 이중 증가 방지)
- XcodeGen 미설치 시 수동 실행 안내 메시지

#### (3) `scripts/install-hooks.sh` — post-merge 설치 추가

기존 pre-commit hook 설치 블록 뒤에 동일 패턴으로:
- 플러그인에서 `post-merge.sh` 복사 (없을 때만)
- `.git/hooks/post-merge` symlink 설치
- 기존 hook 존재 시 안전 건너뛰기

#### (4) `templates/CLAUDE.md.tmpl` §2 — 가드레일 추가

§2 횡단 가드레일에 `🚫 빌드번호 이중 증가 방지 (XcodeGen 프로젝트)` 블록 추가:
- 수동 증가 금지 (hook이 유일한 경로)
- merge 커밋 시 빌드번호 불변 (정상 동작)
- worktree는 main repo hook 공유
- post-merge hook 설명

#### (5) `templates/rules/RULES_VERSIONING.md.tmpl` — 금지 항목 추가

🚫 금지 섹션에 "수동 증가 금지" 항목 추가 (이중 증가 근거 명시).

### 2.3 설계 원칙

| 원칙 | 적용 |
|---|---|
| 단일 책임 | 빌드번호 증가 = pre-commit §(6)만. post-merge = xcodeproj 재생성만 |
| 조건부 활성화 | `CURRENT_PROJECT_VERSION` 존재 시에만 발동. XcodeGen 미사용 프로젝트에 영향 없음 |
| 경로 자동 탐색 | `iOS/`, 루트, `app/` 순서. 모노레포 대응 |
| 이중 증가 원천 차단 | CLAUDE.md + RULES_VERSIONING에 "수동 증가 금지" 이중 명시 |
